魔法函数

__getitem__

会尝试从0开始，一只尝试去循环，直到抛出异常，就结束。

如果在这个魔法函数中没有一些结束循环的异常，就会无限循环下去。

当我们尝试着去循环着对象的时候，python解释器就会尝试去调用这个魔法函数，直到抛异常，就会结束。

当我们定义这个魔法函数的时候，这个普通的类就会变成可迭代类型，如果我们尝试去掉这个魔法函数，就会抛出 TypeError: 'Person' object is not iterable 这样的错误。

一开始，进行循环的时候，python的解释器会先尝试去拿这个类中的 __iter__，如果没有，才会去看 __getitem__ ，这个是python语法的一种优化。

类属性和实例属性以及查找顺序

python 解释器在进行查找的时候，是一个自下而上的顺去去查找变量的。

假如是这样一种继承模式。

在python3中，对于这种继承方式的话，采用的是MRO算法。（表示方法查询的顺序）

深度优先搜索算法：DFS: A->B->D->C->E

但是，如果下面这种方式还采用深度优先搜索算法的话，就会有问题。

如果是这样的话，如果我们在 B 中查询不到想要的方法的话，就会直接去 D 中查找，而事实是，我们在 C 中有实现这个方法，所以按照这种深度优先搜索的话，我们就会失去一个查找点，也就是 C 的覆盖效果就达不到了。

所以在 Python2 之后，就修改了这个算法，及变成 BFS: A->B->C->D

不过这样的话，第一种就会有问题，假如 D 和 C 中有同名的方法，按照广度优先搜索的话，顺序应该是：A->B->C->D->E ，不过按照正常来说的话，如果在 B 中找不到对应的函数的话，就应该在 D 中去寻找，所继承的内容应该是 D 的才对，所以这一种广度优先的算法也不适用。

最后，在 python2.3 之后采用的是一种 C3 算法。（属性搜索算法）并且一直沿用至今。

数据封装和私有属性

在 python 中使用双下划线表示私有属性。

这个时候，如果使用了私有属性的话，就不能直接使用 user.__birthday 来获得数据。会报错说没有这个属性。无法通过实例或者自类获得私有属性。如果在函数前面加上双下划线也可以实现相同的效果。

With 上下文管理器

python 中的 try except 等：

这是捕捉异常的模块，其中的 else 在代码没有出错，也就是没有抛出异常的时候才会执行，而 finally 中的代码在出错和没有出错都会执行。

这个时候，最后函数返回值只有 4 ，一开始，函数在返回 1 之前就异常了，之后被第二个异常所捕捉，在执行完输出语句 print('key error') 之后，放回 2 的时候并没有直接返回，而是将所需要返回的数字 2 放在堆栈中，之后调用 finally 的的返回，同样压在堆栈中，在结束这个异常之后，从堆栈中取出第一个数返回。

所以在python异常处理中，如果有 finally 的话，就会调用finally 的返回，不然就会调用之前的返回。

其中，可以采用python上下文管理器协议，其中涉及到两个魔法函数，__enter__ 和 __exit__ 。

在一个类中，我们在一个类中只要定义了这两个魔法函数，就可以在 with 中使用，可以灵活的处理获得资源和释放资源相关的操作。

contextlib 简化上下文管理器

contextlib.contextmanager 这个装饰器可以将一个函数变成上下文管理器。

使用这个装饰器的话，必须要使用生成器 yield ，可以这么理解，在 yield 之前的可以看成是 __enter__ 要执行的操作，在 yield 之后的，是 __exit__ 所执行的内容。

之所以能实现这样的操作，要归功于生成器的特性。

list 中 extend 方法区别

也可以使用 += 的形式增加列表。

这两种形式的差别就是，采用 + 的时候，两边要求是同一个类型。

而这一种写法只要求是可迭代类型的数据都行。

其实 += 这一种写法是调用了一个魔法函数， __iadd__ ，在这个魔法函数中，会采用 extend 的方法，将数据添加进去。

extend 与 append 也不同，前者会将可迭代的对象循环取出加到 list 中，后者将数组变成一个值，而不是去迭代，直接加上去。

这个注意一下，extend 方法没有返回值。

实现可切片的对象

模式 [start:end:step]

其中，第一个数字start表示切片开始位置，默认为0；
第二个数字end表示切片截止（但不包含）位置（默认为列表长度）；
第三个数字step表示切片的步长（默认为1）。
当start为0时可以省略，当end为列表长度时可以省略，
当step为1时可以省略，并且省略步长时可以同时省略最后一个冒号。
另外，当step为负整数时，表示反向切片，这时start应该比end的值要大才行。

切片的操作会返回一个新的列表，不会改变原来的数组。

实现一个支持切片操作的对象。不可修改序列。
