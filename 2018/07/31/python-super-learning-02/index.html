dict 和 set 的实现原理

可以做测试（在大数据中查找元素），得出。

  • dict查找的性能远远大于list
  • 在list中随着list数据的增大 查找时间会增大
  • 在dict中查找元素不会随着dict的增大而增大

在 dict 中采用的是哈希的方法，所以要求dict的key是一个可哈希的值。

  • dict的key或者set的值 都必须是可以hash的
  • 不可变对象 都是可hash的， str， fronzenset， tuple，自己实现的类 __hash__
  • dict的内存花销大，但是查询速度快， 自定义的对象 或者python内部的对象都是用dict包装的
  • dict的存储顺序和元素添加顺序有关
  • 添加数据有可能改变已有数据的顺序

对象引用、可变性和垃圾回收

变量

python和java中的变量本质不一样，python的变量实质上是一个指针，指向 int 、 str 等变量。

这个时候，应该是先生成 1 或 “abc” 这两个内存空间，这两个内存空间就有对应的类型，之后，a 是作为一个指针，指向这个内存空间，本质上来说，一开始生成的 1 或 “abc” 就是一个有类型的对象了，a 只是作为一个标签的作用，给这个对象贴上标签。这个与 jave 或 c 有很大的差别，对于 java 或 c 来说，应该是在创建这个，类型的变量的时候，就会在内存中申请这个类型的空间大小，相当于一个空盒子，之后在将数据赋值到这个内存中。

其实这样也能看出，a 和 b 只是作为一个名称表示，a 和 b 这两个变量名称所指向的是同一个内存空间，所以当我们改变其中内存空间对象的时候，a 和 b 所反应出来的现象就是内容的改变。

迭代器和生成器

迭代协议

什么是迭代协议。

迭代器是什么？ 迭代器是访问集合内元素的一种方式，一般用来遍历数据。

迭代器和以下标的访问方式不一样，迭代器是不能返回的，迭代器提供了一种惰性方式数据的方式。

从源码可以看出：（在 __collections_abc.py 文件中可以找到）

Iterable 这个类实现的了元类 ABCMeta ，并且类中定义了 __iter___ 这个函数，所以就可以实现迭代，就能获取一条条数据。

Iterator 继承了 Iterable ，并且实现了 __next__ ，所以我们才可以在 for 循环中，获取一条条的数据。

对于一般的可迭代类型来说，只要实现了 __iter__ 这个魔法函数，就是一个可迭代类型。如果要实现一个迭代器，就需要实现 __next__ 方法。例如： list 只实现了 __iter__ 方法，没有实现 __next__ 方法，所以， list 是一个迭代类型，但不是属于一个迭代器。

迭代器和可迭代对象

当我们使用 for 循环的时候，Python 会先尝试去调用 iter(company) （这个内置的函数），而这个函数会首先去寻找类中有没有定义 __iter__ 这个魔法函数，当有这个魔法函数的时候，就会直接使用这个魔法函数所实现的内容，如果类中没有实现，这个函数就会创建一个默认的迭代器，之后迭代器就会利用 __getitem__ 函数进行迭代（并且从0开始遍历）。

生成器原理

首先看 Python 中函数的工作原理

我们执行的时候，是 python.exe 在执行的，是采用 C 语言编写的函数，这个 exe 文件会去执行一个叫做 PyEval_EvalFramEx (c函数)去执行 foo 函数， 首先会创建一个栈帧 (stack frame 上下文)。

python一切皆对象，栈帧对象， 会把代码变成字节码对象

当foo调用子函数 bar， 又会创建一个栈帧

所有的栈帧都是分配在堆内存上，这就决定了栈帧可以独立于调用者存在

我们可以通过 dis 这个对象来查看字节码对象

这个是字节码的过程，加载全局 (LOAD_GLOBAL)，把 b 放进来，之后调用这个函数（CALL_FUNCTION），调用这个字节码，从栈的顶端把这个元素打印出来，之后 LOAD_CONST， 函数没有 return 所以返回一个 None，之后把 None return 回来。

这个时候，foo 函数执行玩了，但是对于全局变量来说，拿到了在执行过程中 bar 的栈帧。最后在输出的时候，通过栈帧对象，拿到了 code 对象，之后在这个对象中拿到了这个函数的名称。

通过 f_back 就可以拿到调用他（当前栈帧）的栈帧。

通过这个例子我们可以看出，当函数执行完之后，函数并没有被销毁，而是还保留在在堆栈中，我们依旧可以拿到栈帧对象，这个与静态语言有着很大的不同，静态语言在调用的时候采用一种栈的形式，这时候，一旦函数执行完毕，这个栈就会被销毁。

frame 栈帧：

  • f_back: 调用栈的前一帧。
  • f_code: 栈帧对应的code对象。
  • f_locals: 用在当前栈帧时与内建函数locals()相同，但你可以先获取其他帧然后使用这个属性获取那个帧的locals()。
  • f_globals: 用在当前栈帧时与内建函数globals()相同。

code 对象：

  • co_argcount: 普通参数的总数，不包括*参数和**参数。
  • co_names: 所有的参数名（包括*参数和**参数）和局部变量名的元组。
  • co_varnames: 所有的局部变量名的元组。
  • co_filename: 源代码所在的文件名。
  • co_flags: 这是一个数值，每一个二进制位都包含了特定信息。较关注的是0b100(0x4)和0b1000(0x8)，如果co_flags & 0b100 != 0，说明使用了*args参数；如果co_flags & 0b1000 != 0，说明使用了**kwargs参数。另外，如果co_flags & 0b100000(0x20) != 0，则说明这是一个生成器函数(generator function)。
